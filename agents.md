# Agents 规则

## index.md 索引文件

1. `index.md` 是本项目文件夹的完整索引，描述目录结构和每个文件的内容概要。

## 更新索引

当用户指示"更新索引"时：

1. 通过 `git diff HEAD --name-status` 或 `git status` 找到自上一次索引版本以来的文件变更（新增、修改、删除、重命名）。
2. 基于变更差异，更新 `index.md` 中的目录结构和文件清单。
3. 将当前最新 commit hash 写入 index.md 顶部的「索引版本」字段。

## 建立索引

如果 `index.md` 不存在：

1. 完整扫描项目目录下的所有文件和文件夹。
2. 建立完整的目录结构树和每个文件的简要说明。
3. 将索引写入 `index.md`，并附上当前 commit hash。
